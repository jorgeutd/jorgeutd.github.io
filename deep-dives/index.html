Papers & Deep Dives

How LLMs actually work

The interesting part of an LLM isn’t that it predicts the next token — it’s how you shape that prediction into useful text, and the papers that made it fast enough to serve at scale. Below: a live, honest decoding playground, then short explainers of the work behind modern inference.

Interactive — runs entirely in your browser; no data leaves the page.

1 · How a model picks the next token

A model outputs a logit for every token in its vocabulary. Softmax turns those into probabilities; temperature sharpens or flattens them, and top-k / top-p (nucleus) trim the long tail before a token is sampled. Try it on this example distribution for the prompt “The capital of France is ___”.

2 · How a model knows token order

Attention is permutation-invariant — on its own it has no sense of order. Sinusoidal positional encodings give every position a unique, smoothly varying signature built from sines and cosines at geometrically spaced frequencies: PE(pos,2i)=sin(pos / 10000^(2i/d)), PE(pos,2i+1)=cos(…). Each row is a position, each column a dimension. (RoPE, in the papers below, rotates query/key vectors by these angles instead of adding them.)

−1 0 +1 — low-dimension columns oscillate fast, high-dimension columns drift slowly, so every position gets a unique fingerprint the model can learn to read.

3 · The papers behind modern inference

Plain-English summaries of the work I lean on daily — what each one changed, and why it matters in production.

Attention Is All You Need Vaswani et al., 2017

TL;DR: replace recurrence with self-attention — every token attends to every other in parallel.

The Transformer. Parallelizable, scalable, and the substrate every modern LLM is built on. Everything below exists to make this operation cheaper at inference time.

arXiv:1706.03762 →

FlashAttention Dao et al., 2022 (v2/v3 since)

TL;DR: exact attention, but IO-aware — tile the computation in on-chip SRAM to avoid round-trips to slow HBM.

Same math, far fewer memory reads/writes — big speedups and longer contexts for both training and inference. FlashAttention-3 targets Hopper/Blackwell.

arXiv:2205.14135 →

PagedAttention / vLLM Kwon et al., 2023

TL;DR: store the KV cache in fixed-size pages like virtual memory instead of one contiguous slab.

Kills internal fragmentation and lets sequences share blocks, so a GPU serves far more concurrent requests. Now the default memory model across serving stacks.

arXiv:2309.06180 →

Speculative Decoding Leviathan / Chen, 2023; EAGLE, 2024

TL;DR: a small draft model proposes several tokens; the big model verifies them in one parallel pass.

Provably the same output distribution, but fewer expensive forward passes — a major latency win. EAGLE-2/3 are the dominant production variants in 2026.

arXiv:2211.17192 →

Multi-head Latent Attention DeepSeek-V2/V3, 2024

TL;DR: compress keys/values into a low-rank latent before caching, instead of sharing heads (GQA).

Shrinks the KV cache dramatically while keeping quality, enabling long context at lower memory. The reason the KV-cache formula doesn’t apply cleanly to DeepSeek models.

arXiv:2405.04434 →

RadixAttention / SGLang Zheng et al., 2024

TL;DR: keep cached KV prefixes in a radix tree so any shared prefix is reused automatically.

Turns overlapping system prompts, few-shot preambles, and multi-turn history into cache hits — large throughput gains for agents and RAG.

arXiv:2312.07104 →

LoRA & QLoRA Hu et al., 2021; Dettmers et al., 2023

TL;DR: freeze the base model and train tiny low-rank adapters; QLoRA does it on a 4-bit-quantized base.

Fine-tune large models on a single GPU with a fraction of the memory — how I adapt open models (Llama, Gemma, DeBERTa) for function calling and domain tasks.

arXiv:2305.14314 →

Rotary Position Embeddings (RoPE) Su et al., 2021

TL;DR: encode position by rotating query/key vectors by an angle proportional to position, rather than adding a positional vector.

Relative position falls out of the dot product for free, and it extrapolates to longer contexts (with scaling like YaRN). Used by Llama, Qwen, Mistral, and most modern open models.

arXiv:2104.09864 →

Mixture-of-Experts (Mixtral) Jiang et al., 2024

TL;DR: route each token to a few expert FFNs out of many, so total parameters far exceed the active parameters per token.

Big-model quality at small-model inference compute — the design behind Mixtral, Qwen3-235B-A22B, gpt-oss, and DeepSeek. Attention/KV still dominates memory; MoE cuts the compute.

arXiv:2401.04088 →
